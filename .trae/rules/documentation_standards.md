## 文档头规范

所有正式文档类输出（非 README/CLAUDE/rules 等元文档）必须包含文档头：

| 字段 | 说明 |
|------|------|
| 文档标题 | 文档的完整标题 |
| 文档作者 | 文档编写者 |
| 文档日期 | 编写/修改日期 (YYYY-MM-DD) |
| 文档版本 | 语义化版本号 (vX.Y) |
| 文档描述 | 一句话说明文档用途 |

标准格式：

```markdown
| 属性 | 内容 |
|------|------|
| **文档标题** | xxx |
| **文档作者** | xxx |
| **文档日期** | YYYY-MM-DD |
| **文档版本** | v1.0 |
| **文档描述** | xxx |
```

## 版本号管理规则

| 规则 | 说明 |
|------|------|
| 初始版本 | v1.0 |
| 评审修复（阻塞项） | minor 版本递增（v1.0 → v1.1） |
| 评审修复（仅建议项） | 可不递增版本号，但修订历史中需记录 |
| 跨文档同步触发 | 被同步文档必须递增 minor 版本（至少 minor 版本） |
| 重大重构 | major 版本递增（v1.x → v2.0），设计文档阶段一般不涉及 |

## 修订历史格式

所有正式文档必须包含修订历史章节，位于文档末尾：

```markdown
## 修订历史

| 版本 | 日期 | 修改内容 | 作者 |
|------|------|---------|------|
| v1.0 | YYYY-MM-DD | 初始版本：简要描述 | 作者 |
| v1.1 | YYYY-MM-DD | 评审修复：B-1 修复说明；S-1 修复说明 | 作者 |
```

**修订内容编写规则**：

| 场景 | 格式 | 示例 |
|------|------|------|
| 初始版本 | `初始版本：关键决策摘要` | `初始版本：定义 AgentRegistration 结构和注册流程` |
| 评审修复 | `评审修复：B-N 修复说明；S-N 修复说明` | `评审修复：B-1 拆分 AgentRegistration 为 Spec+Runtime` |
| 跨文档同步 | `同步更新：同步来源和内容` | `同步更新：新增 ARTIFACT_TYPE_EXECUTION_RECORD（来自 Task 3）` |

## 跨文档引用格式

| 引用类型 | 格式 | 示例 |
|---------|------|------|
| 同目录文件 | `[文件名](文件名)` | `[task_artifact_data_model.md](task_artifact_data_model.md)` |
| 上级目录文件 | `[文件名](../文件名)` | `[architecture.md](../architecture.md)` |
| 项目根目录文件 | `[文件名](../../文件名)` | `[README.md](../../README.md)` |

## Markdown 格式规范

- 文档必须使用 CommonMark 规范的 Markdown 语法
- 表格用于结构化数据呈现
- 代码块必须标注语言类型
- `#` 仅用于文件主标题，章节正文使用 `##` / `###` 层级
- 列表使用 `-` 而非 `*`

## 日志规范

所有日志类输出使用中文，格式为：

```
[日志级别] [YYYY-MM-DD HH:MM:SS] 日志内容
```

日志级别：`DEBUG` / `INFO` / `WARN` / `ERROR` / `FATAL`
